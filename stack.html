JavaScript Performance

General:
- Limit the amount of data shown. e.g. pagination, infinite-scroll etc.
- Use lightweight components / libraries.
- Avoid animations, gradients, dom manipulations.
- Remove non-applicable / debug code in production

Angular-specific:
- Use single binding instead of two-way (e.g. bindonce).
- Pre-computed values at template level.
- Avoid polluting scopes.
- Use of $digest instead of $apply. And $emit instead of $broadcast.
- Role-based access.
- Composition over inheritance. Simple objects instead of mixins.
- Wrappers on common angular api.
- Use ng-if / ng-switch instead of ng-show / ng-hide.
- Use "track by $index" in ng-repeat (dom re-use).
- Use $parse instead of $eval.
- Directives: Initialize expressions in "compile" and then use in "link" function.
- Use $watchExpression instead of $watch.
